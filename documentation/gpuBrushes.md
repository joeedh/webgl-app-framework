# GPU brushes (kelvinlet / grab)

When dynamic topology is **off**, the kelvinlet and grab sculpt brushes can run
entirely on the GPU: the compute kernels dispatch from the TS host app on the
**renderer's own WebGPU device**, deformed positions scatter straight into the
render vertex buffers, and the CPU mesh is read back once, at stroke end (for
the C++ mesh + undo). Design + milestones:
[plans/gpuGlobalBrushes.md](plans/gpuGlobalBrushes.md). This works identically
on both backends (browser/WASM and NW.js/N-API) because the app renders on the
TS-side WebGPU device under both.

At 5 M tris the CPU kelvinlet costs ~500 ms/dab; the GPU path's steady-state
dab is ~2 ms CPU + ~1.5 ms GPU (the one-time stroke-begin cost — geometry
upload + undo snapshot — is ~750 ms at that size; see the M5 result in the
plan).

## Flags

- **`sculptcore.gpu_brush`** (default **on**) — run eligible brushes on the
  GPU. Eligibility is decided once per stroke (never mid-stroke): dyntopo off,
  no autosmooth chain, tool in the GPU kernel map, WebGPU device present. Any
  miss falls back to the CPU path silently.
- **`sculptcore.gpu_brush_grab`** (default **off**, soak) — additionally let
  the grab brush take the GPU path.
- **`sculptcore.gpu_brush_verify`** (default off) — shadow-verify (§ below).

Headless `exec()` replay of saved strokes always uses the CPU path (replay
determinism outweighs replay speed).

## Architecture (one paragraph per layer)

- **Kernels** are the sbrush WGSL emits, shipped as the committed
  `sculptcore/typescript/sculptcore/brush/brushWgsl.ts` (regenerated by
  `node make.mjs codegen`). Grab-class brushes are tagged `@grabmode` in their
  `.sbrush`: the WGSL deforms from the stroke-start positions (binding 22) and
  write-back does per-dab first-touch arbitration via a dab-stamp buffer
  (binding 23) — the GPU twin of `AccumOrigGrab`/`grabClaimFirstTouch`.
- **Marshaling** is C++-owned (`sculptcore/source/brush/gpu_marshal.{h,cc}` +
  the `GpuBrush_*` seam in `gpu_brush_session.h` / `c-api/gpu_brush_c_api.cc`):
  the app uploads opaque blobs packed per `compute_layout.h` and never
  re-derives a layout. The seam also takes the undo snapshots (into the open
  MeshLog step) and applies the stroke-end write-back.
- **Dispatch** is `scripts/webgpu/brush_compute.ts` (`GpuBrushStroke`):
  bind-group layouts are introspected from the kernel's `@binding(n)` table
  (an unknown binding fails loudly at stroke begin), per-dab uploads are
  uniforms + LUT only once the filter set saturates, and a scatter kernel fans
  positions + flat-shaded face normals into each touched GPU node's VBOs
  (`fill_leaf_slice` parity — corner→vert maps come from
  `SpatialTree::buildGpuScatterTables`, cached across strokes on
  `SpatialTree::gpuLayoutGen`).
- **Stroke orchestration** is
  `scripts/editors/view3d/tools/sculptcore_gpu_stroke.ts`
  (`GpuStrokeController`), owned by `SculptPaintOp`: eligibility, per-dab
  marshal→dispatch, the serialized async chain (readbacks, stroke-end apply,
  undo requests arriving mid-await), and the debug surface.

During a GPU stroke the CPU mesh is **intentionally stale** (D4): raycasts hit
the stroke-start surface until the stroke ends (or, in the M2 fallback shape,
until a coalesced readback lands). Undo/redo work exactly as on the CPU path —
one MeshLog step per stroke, snapshots taken before any write.

## Debugging (§9 of the plan)

Surface: **`window.DEBUG.gpuBrush`** (also `CTX.debug.gpuBrush`), reachable
over CDP per [debugSurface.md](debugSurface.md):

- `state()` — live session dump (kernel, counts, last uniform blobs as hex).
- `forceReadback()` — sync the CPU mesh from the GPU mid-stroke so every
  existing inspection tool sees live state.
- `capture(n)` — record the next n GPU strokes as
  `sculptcore/tests/webgpu/replay.mjs` fixtures (`lastFixture`), directly
  replayable through Dawn bit-exact.
- `scatterSelfCheck()` / `selfCheckNext` — byte-check one node's scattered VBO
  against a CPU gather over the corner→vert map.
- `allowNonModal` — test hook: lets non-modal (execTool-driven) strokes take
  the GPU path (headless drivers).
- `verbose` — `[gpu-brush]` per-dab logging.

Always-on hatches: a **non-finite tripwire** kernel ORs a NaN/Inf flag over
each dab's work set (async readback; on trip the stroke finishes conservatively
and never writes poisoned data to the mesh/undo), and every buffer/pipeline/
pass is labeled `gpuBrush.<kernel>.<purpose>` with error scopes around setup +
the first dab and a device-lost handler that flips to the CPU-fallback finish.

**Shadow-verify** (`sculptcore.gpu_brush_verify`): every dab runs on both
paths — the CPU executor stays authoritative, the GPU runs in parallel, and
the dab's work set is diffed per vertex. On breach it logs the first
divergent verts, auto-captures the fixture, and re-syncs the GPU from the CPU
so divergence never compounds. First tool to reach for on any "GPU stroke
looks wrong" report.

The HUD line on the sculptcore toolmode (`gpuBrushStatsLabel`) mirrors
`dynTopoStats`: dabs/dispatches/scatter passes, filter-set size, upload bytes,
marshal/upload/submit CPU ms, readback ms, and GPU pass ms when the adapter
grants `timestamp-query`.

## Troubleshooting (symptom → tool)

| Symptom | Tool |
|---|---|
| Wrong deformation shape | `sculptcore.gpu_brush_verify` (shadow-verify) — divergent verts + auto-captured fixture split "wrong bytes uploaded" from "kernel computed wrong values" (replay the fixture through `replay.mjs`) |
| Wrong shading / corrupted-looking surface | `DEBUG.gpuBrush.scatterSelfCheck()` — scatter-map/fill-order disagreement shows as bad corners |
| Explosion / NaN spray | Non-finite tripwire (check `lastStats.tripwireTripped`; the auto-capture has the offending dab's exact bytes) |
| Stroke silently on CPU | `DEBUG.gpuBrush.state()` — `active:false` means eligibility declined (dyntopo on? autosmooth? flag off? grab without `gpu_brush_grab`? no device?) |
| Slow | HUD `gpuBrushStatsLabel` / `lastStats` — marshal vs upload vs submit vs GPU ms; see the M5 notes in the plan |
| Undo looks wrong | The §8.3 gates in `tests/integration/sculptcore_gpu_brush.test.ts` bracket this; `forceReadback()` + vert dumps inspect live state |

## Tests

`tests/integration/sculptcore_gpu_brush.test.ts` (both backends, self-skipping)
drives `__gpuBrushTest()`: world-space CPU-vs-GPU parity at fp tolerance with
X symmetry (kelvinlet + grab), exact undo/redo, zero shadow-verify
divergences, a clean scatter self-check, and a captured fixture replayed
bit-exact through Dawn. The kernel-level gates are `make.mjs sbrush-verify`,
`webgpu-verify`, and `wgpu-native-verify` (22 brushes each). See
[debugStrokeGuide.md](debugStrokeGuide.md) for driving GPU strokes headlessly.
